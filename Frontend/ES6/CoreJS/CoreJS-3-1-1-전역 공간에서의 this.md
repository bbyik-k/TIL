# [Core JS] 3-1-1. 전역 공간에서의 this

## 요약
전역 공간에서의 this는 실행 환경에 따라 전역 객체를 가리키며, 전역 변수와 전역 객체 프로퍼티의 관계를 이해하는 것이 디버깅과 예측 가능한 코드 작성에 중요하다.

## 핵심 내용

### 핵심내용 1: 전역 공간에서 this의 대상
자바스크립트에서 this는 실행 컨텍스트가 생성될 때 함께 결정된다. 전역 실행 컨텍스트에서 this는 항상 전역 객체를 참조한다.  
브라우저 환경에서는 window, Node.js 환경에서는 global이 그 대상이 된다.

### 핵심내용 2: 전역 변수와 전역 객체의 연결 방식
전역에서 var로 선언한 변수는 전역 객체의 프로퍼티로 등록된다.  
이 때문에 전역 공간에서 변수 이름으로 접근하든, this.변수명 혹은 전역객체.변수명으로 접근하든 동일한 값을 참조하게 된다.

### 핵심내용 3: 선언 방식에 따른 차이
전역 변수 선언과 전역 객체에 직접 프로퍼티를 추가하는 행위는 결과가 비슷해 보이지만 완전히 동일하지는 않다.  
var로 선언한 전역 변수는 delete로 제거할 수 없지만, 전역 객체에 직접 추가한 프로퍼티는 삭제가 가능하다.

## Check Point
- 전역 실행 컨텍스트에서 this는 무엇을 가리키는가?
- 브라우저와 Node.js 환경에서 전역 객체는 각각 무엇인가?
- var 전역 변수와 window 프로퍼티는 완전히 같은 개념인가?

## 언제 문제가 되는가 / 언제 쓰는가
디버깅 중 전역 변수 충돌이나 예상치 못한 값 변경이 발생할 때, this가 전역 객체를 가리킨다는 사실을 모르면 원인을 추적하기 어렵다.  
특히 오래된 코드나 스크립트 기반 환경에서 전역 변수 오염 문제를 분석할 때 이 개념이 직접적으로 사용된다.

## 정리
전역 공간에서의 this는 전역 객체 그 자체이며, 전역 변수 동작을 이해하는 기준점이 된다.
