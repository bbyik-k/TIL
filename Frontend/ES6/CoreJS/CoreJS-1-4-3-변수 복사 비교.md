# [Core JS] 1-4-3. 변수 복사 비교

## 요약

- 변수 복사는 모두 _주소값의 복사_ 이지만, 기본형과 참조형은 그 이후의 변화 방식이 완전히 다르다.
- 이 차이를 이해하지 못하면 값 변경, 상태 관리, 디버깅에서 예측 불가능한 버그를 만든다.

## 핵심 내용

- 변수 복사는 기본형과 참조형 모두 동일한 방식으로 시작된다.
- 차이는 **_복사 이후 값이 변경될 때_** 발생한다.
- 참조형의 가변성은 객체 자체가 아니라 내부 프로퍼티 변경에서만 성립한다.

### 핵심내용 1

기본형과 참조형 모두 변수를 복사할 때는 주소값을 복사한다.  
기본형은 값이 저장된 주소를, 참조형은 객체가 저장된 주소를 변수 영역에 복사한다.  
즉, 복사 시점까지의 동작은 두 타입 모두 동일하다.

### 핵심내용 2

복사 이후 값이 변경될 때 차이가 발생한다.  
기본형은 새로운 값이 필요하면 데이터 영역에 새 값을 만들고, 변수는 새로운 주소를 가리킨다.  
반면 참조형은 객체 내부 프로퍼티를 수정하면 기존 객체를 그대로 사용하므로, 복사된 다른 변수에도 변경이 전파된다.

### 핵심내용 3

참조형이 “가변”이라고 불리는 이유는 객체 자체가 바뀌기 때문이 아니다.  
같은 객체를 참조한 상태에서 내부 프로퍼티를 수정할 수 있기 때문이다.  
객체 자체를 새로 할당하면, 그 순간에는 기본형과 동일하게 동작한다.

## Check Point

- 변수 복사는 값 복사가 아니라 주소 복사라는 점을 이해했는가?
- 참조형의 가변성이 언제 발생하는지 명확히 구분할 수 있는가?
- 객체 프로퍼티 변경과 객체 재할당을 혼동하고 있지는 않은가?

## 언제 문제가 되는가 / 언제 쓰는가

- React state에서 객체를 직접 수정하면 이전 state와 같은 객체를 참조하게 되어 렌더링이 발생하지 않는다.
- 디버깅 시 “값을 하나만 바꿨는데 다른 변수도 같이 바뀌는 현상”의 원인

## 정리

변수 복사의 핵심 차이는 복사 순간이 아니라, 복사 이후 값을 어떻게 변경하느냐에 있다.
