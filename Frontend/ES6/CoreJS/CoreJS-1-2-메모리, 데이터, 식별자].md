# [Core JS] 1-2. 메모리와 데이터 그리고 식별자와 변수

## 요약

컴퓨터에서 모든 데이터는 메모리에 저장되며,  
**데이터를 직접 다루는 것이 아니라 메모리 주소를 통해 간접적으로 접근**한다.
식별자(변수명)는 이 주소를 사람이 다루기 위한 이름이다.

## 핵심 내용

### 1. 메모리와 데이터

- 컴퓨터는 모든 데이터를 0과 1로만 표현한다.
- 1비트(bit)는 0 또는 1 하나만 표현할 수 있는 최소 단위다.
- 비트 단위로 데이터를 관리하면 비효율적이기 때문에,
  여러 비트를 묶은 **바이트(byte)** 단위가 등장했다.
- 1바이트는 8비트로 구성되며, 총 256(2⁸)개의 값을 표현할 수 있다.
- 바이트 단위로 구분된 각 메모리 공간은
  **고유한 식별자**, 즉 메모리 주소(memory address)를 가진다.
- 모든 데이터는 이 **메모리 주소를 기준으로 구분되고 연결**된다.

### 2. 식별자와 변수

- 숫자, 문자열, 객체, 배열 등은 모두 데이터다.
- 이 데이터를 직접 다루는 것이 아니라,
  **어떤 데이터를 가리키는지를 식별하기 위해 이름을 붙인다.**
- 이때 사용하는 이름이 **식별자(identifier)** 다.
- 변수(variable)는
  - 변할 수 있는 수가 아니라
  - **변할 수 있는 데이터를 가리키는 식별자**다.
- 즉,
  - 데이터는 메모리에 있고
  - 변수(식별자)는 그 메모리 주소를 가리킨다.

## Check Point

- “값을 저장한다”는 표현이 실제로는 무엇을 의미하는지

## 언제 문제가 되는가 / 언제 쓰는가

- 변수에 값을 저장한다고 생각했는데, 실제로는 값이 아닌 **주소를 다루고 있다는 점을 놓칠 때**
- 기본형과 참조형의 차이를 메모리 관점에서 이해하지 못할 때
- 변수 복사, 불변성, 가변성 개념이 헷갈릴 때
- 이후 실행 컨텍스트, 스코프, 클로저를 이해할 때

## 정리

자바스크립트에서 데이터를 다룬다는 것은  
**메모리 주소를 통해 값에 접근하는 구조를 이해하는 것**부터 시작.  
데이터 → 메모리 주소 → 식별자
